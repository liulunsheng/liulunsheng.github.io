实验三 使用Github管理Android项目

一、实验目的

  1. 掌握Android Studio中Git使用
  2. 掌握Github管理项目

二、实验任务

  1. 在Android Studio中新建项目
  2. 添加VCS中的Git支持
  3. 发布项目到Github
  4. 在Android Studio中进行修改代码、提交、新建分支、合并等操作
  5. 推送项目到Github
  6. 如有多人合作，处理pull request

三、实验步骤与结果

  1. 在Android Studio里新建项目，我的项目是美食推荐APP，所以取名为FoodRecommendation。
  2. 添加VCS中的Git支持。点击菜单栏VCS—>Import into Version Control—>Share Project on Github，然后填写自己的github账号和密码，如下图所示

​ 添加完Git后，VCS栏中多了Git选项如图所示

  1. 发布项目到Github。点击Git—>Push，即可发布到Github上。发布完成后如下图所示

  1. 在Android Studio里修改代码。这里我在activity_food.xml中添加了三个下拉菜单，同时，可以看到相应的被改过的文件变成了蓝色。提交时，点击Commit Changes，填写好Commit Message，点击Commit and Push，即可提交更改。

四、实验小结

在Android Studio里添加Git后，更加方便了上传更改的项目，使原先复杂的过程集成到一起，简单又方便。如果本地文件丢失，还能从github上恢复，等于是给项目加了一个保险。

  8. 保存更改后可以同步到github网页，只需点击即可同步。这时可以看到网页上的commits也更新了，如下图所示

  9. 如果要对代码进行在原基础之上的修改，又不想破坏源代码，可以选择新建一个分支，

  10. 修改readme.txt内容并保存，并将bug2这个分支合并到master分支上，即可对代码进行更改。可以看到github上branch变成了两个

实验一 在Github上用Markdown创建技术博客

一、实验目的

  1. 熟悉Github网站
  2. 掌握Markdown语法

二、实验任务

  1. 注册Github账号
  2. 创建Gihub Pages repository
  3. 发布第一篇博客
  4. 学习Markdown
  5. 利用Hexo生成博客并部署到Github

三、实验步骤与结果

  1. 注册Github账号

    账号：liulunsheng

  2. 创建Gihub Pages repository

    Repository名：liulunsheng.github.io

  3. 发布第一篇博客，如下图所示

    共同协作修改如下图所示：

  1. 学习Markdown

  2. 利用Hexo生成博客并部署到Github

    • 安装node-v7.7.1-x86以及Git-2.12.0-32-bi

    • 打开https://npm.taobao.org/，复制npm指令。

    • Windows+R，输入cmd,调出窗口，输入之前复制的npm指令。

    • 创建hexo目录，窗口输入

      cnpm install –g hexo-cli

    • 路径定位到D盘DM文件下，输入hexoinit hexo，进入到hexo文件，

    • 输入cnpm install以完成相关配置。

    • 窗口输入hexo server，运行本地测试，打开http://localhost:4000/。

    • 在github中创建一个repository，名称为liulunsheng.github.io。修改hexo文件根目录下的_config.yml文件。

    • 新建博客。输入hexo new testMyBlog 生成一片新的文章，将repository中的md文件复制到_posts文件夹下。

    • 生成博客。窗口输入hexo generate 指令对原有Html文件进行重新生成；输入hexo server 本地预览，输入hexo deploy 指令将Html文件发布到Github Pages服务器中。

  3. 实验结果

四、实验小结

​ 通过学习github以及markdown语言，学会了如何创建自己的博客，也了解了关于markdown语言的一些知识。但还是不够深入，缺乏实践，往后需更加努力。
